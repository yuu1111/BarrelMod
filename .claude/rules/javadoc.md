# JavaDocスタイル

## スタイルルール

1. HTMLタグを使わない (`<p>`, `<ul>`, `<li>`, `<br>` など全て禁止)
2. 必ず3行形式で書く (1行形式 `/** ... */` は禁止)
3. 句点「。」を使わない (文末は体言止めか「する」「である」で終わる)
4. 全角括弧「（）」を使わない (半角 `()` を使用)
5. 括弧による補足説明を避ける (冗長になりやすいため、文を分けるか削除)
6. バナーコメント禁止 (`====`, `----`, `****` などの区切り線)
7. `@param`と`@return`は必須 (引数・戻り値がある場合は必ず記載)
8. インターフェース実装にもJavaDocを書く
9. 説明文内に空行を入れない (連続した文は改行で繋げる)
10. 説明文と@paramの間は空行1つのみ
11. コード例 (`{@code}`, `{@link}` 以外のコードブロック) を書かない
12. 簡潔に記述する

## 変換パターン

| Before | After |
|--------|-------|
| `/** 説明 */` | `/**`<br>` * 説明`<br>` */` |
| `情報を取得する。` | `情報を取得する` |
| `<p>詳細説明` | `詳細説明` (改行で分離) |
| `<ul><li>項目1</li></ul>` | `- 項目1` (プレーンテキストのリスト) |
| `（必須）` | `(必須)` |
| `@return 値。` | `@return 値` |
| `@param message メッセージ({}でパラメータを指定)` | `@param message フォーマット用メッセージ` |
